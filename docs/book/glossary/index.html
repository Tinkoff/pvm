Locators‚Äã

–°The package selector is used in pvm settings to select groups of packages in a monorepository.

There are two kinds of selectors:

Selectors by package path‚Äã

Must start with /. Examples:

/src/core/**/*
/src/utils/*

For all the globbing features that you can use, see here.

Package name Selectors‚Äã

Must not start with /. Examples:

@pvm/*
@pvm/pvm
@pvm/core

Globbing works on the same principles as in the case of paths, except for the option basename ‚Äì it is included so that * matches packets with namespace: i.e. * will match all packages, even if they have a namespace.

Canary release‚Äã

Release packages with a version like 1.1.0-043fdc15.0, where 043fdc15 is the commit from which the release is being made. The version will not be saved in the list releases and will not be included in the changelog.

Canary index‚Äã

In the example version 1.1.0-043fdc15.0, the part after 043fdc15. is an index and is incremented in case of publishing a new canary version with the base version and preid unchanged.
